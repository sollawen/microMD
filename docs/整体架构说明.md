# microNeo 架构总结

> 生成日期: 2026-06-10

## 一、目录结构

```
internal/
├── md/                    # Markdown 渲染引擎
│   ├── md.go              # 核心类型定义
│   ├── detect.go          # 集中式分类器
│   ├── inline.go          # 行内标记处理（加粗、斜体、链接等）
│   ├── wrap.go            # 文本换行
│   ├── render_table.go    # 表格渲染器
│   ├── render_codeblock.go# 代码块渲染器
│   ├── render_heading.go  # 标题渲染器
│   ├── render_list.go     # 列表渲染器
│   ├── render_blockquote.go# 引用块渲染器
│   └── render_hr.go       # 水平分割线渲染器
│
├── display/
│   ├── bufwindow.go       # Display() 加了 if/else 分支
│   └── bufwindow_md.go    # MD 专用显示逻辑
│
├── action/
│   └── bufpane_md.go      # MD 专用事件处理（initMDConfig、editMode 切换）
│
└── buffer/
    └── buffer.go          # 新增 IsMD、MDSegments 字段
```

## 二、核心类型

```go
// Cell 是渲染管线输出的最小单位
type Cell struct {
    Rune         rune
    Style        tcell.Style
    BufLine      int          // 对应 buffer 行号（装饰行=-1）
    BufX         int          // rune 偏移（装饰行=-1）
    IsDecorative bool         // 装饰字符不响应点击
}

// Segment 是检测步骤的输出单位
type Segment struct {
    BufStartLine int
    BufEndLine   int
    VisibleStart int
    VisibleEnd   int
    Render       func(seg Segment, width int, cfg MDConfig) *RenderedSegment
}
```

## 三、渲染管线

### 检测与渲染分离

| 阶段 | 职责 | 触发时机 |
|------|------|----------|
| **detect** | 分类：哪几行对应哪个 render | 事件驱动（buffer 开/编辑） |
| **render** | 布局：lines 铺成 screen cells | 显示驱动（每帧调用） |

### detect.go 分类逻辑

```
扫描 buffer 行：
├── highlighter state 非 nil？→ 代码块 → RenderCodeBlock
├── 以 > 开头？        → 引用块 → RenderBlockquote
├── 以 | 开头？        → 表格 → RenderTable
├── 以 -/*/+ 开头？   → 列表 → RenderList
├── 以 # 开头？       → 标题 → RenderHeading
├── 满足水平分割线？  → RenderHR
└── 其他              → 普通文本 → RenderNormal
```

### displayBufferMD 主循环

```go
func (w *BufWindow) displayBufferMD(editMode bool) {
    // 1. 读取事件驱动的分类结果
    segments := filterSegmentsToVisible(b.MDSegments, visibleStart, visibleEnd)
    
    // 2. 逐 segment 渲染
    for _, seg := range segments {
        if editMode && hasCursorInside(seg, cursors) {
            renderSegmentNative(seg)   // 光标所在 segment 回退原生
        } else {
            renderSegmentMD(seg)       // 其他走 MD 渲染
        }
        // 写入 mdCache（副作用，供 Scroll/Relocate 使用）
    }
}
```

## 四、Buffer 层扩展

```go
type SharedBuffer struct {
    *LineArray
    IsMD       bool              // MD 文件标记（NewBuffer 算一次）
    MDSegments []md.Segment       // 检测分类结果（事件驱动缓存）
}

type BufWindow struct {
    mdCache  []md.SegmentMeta    // 渲染副作用（每帧刷新）
    editMode bool                // 编辑模式标志
    mdConfig md.MDConfig         // MD 配置
}
```

## 五、依赖关系

```
NewBuffer → IsMD
    ↓
MarkModified → DetectSegments → MDSegments
    ↓
Display() → displayBufferMD() → renderSegmentMD() → seg.Render()
```

## 六、架构决策

1. **IsMD 真源在 SharedBuffer** - NewBuffer 时计算一次
2. **检测与渲染分离** - detect 事件驱动，render 显示驱动
3. **每一行都是渲染片** - 统一管线，无分支
4. **渲染器是纯函数** - 只做计算转换，不碰 tcell screen
5. **displayBufferMD 是超集** - editMode 时光标所在 segment 回退原生逻辑
