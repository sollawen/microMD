## 一、项目起因

日常工作中频繁使用 Markdown 文件，遇到一个反复出现的痛点：

1. **Micro 编辑器**里看 Markdown 表格，列没有对齐，每行挤在一起，非常难受
2. 退出来改用 **Leaf** 查看表格，表格好看了，但又不能编辑
3. 要编辑只能退回 Micro，来回切换很烦

**核心需求**：在 Micro 里编辑 Markdown 时，表格能自动按列对齐显示，其他功能不变。

### 改造思路

核心思路：**只改 display 层，不动 buffer 层**

1. **检测表格块** — 连续几行都以 `|` 开头和结尾，就是表格
2. **计算列宽** — 每列按最宽内容对齐补空格
3. **渲染对齐版** — 送到屏幕显示
4. **光标映射** — 屏幕坐标 ↔ buffer 坐标的双向转换

预计改动量：**~500 行**，主要集中在 `internal/display/bufwindow.go`。

| **Kiro** | Rust | 761 | ❌ | 新兴编辑器 |

### 3.2 关键发现

> **所有能编辑的终端编辑器，没有一个能正确渲染 Markdown 表格。**
> **唯一能渲染表格的 Leaf 和 Glow，都是只读查看器，不能编辑。**

市场空白位置：

```
                  能编辑  |  不能编辑
              ───────────┼───────────
能渲染表格    【我们的机会】 |   Leaf / Glow
不能渲染表格  Micro/Helix  |   bat / glow CLI
```

### 3.3 重点竞品分析

#### Helix（44K⭐）
- **不构成竞争**：模态编辑，学习曲线陡，目标用户是硬核开发者
- 走的是 LSP + tree-sitter 路线，和我们的轻量定位完全不同

#### Lapce（38K⭐）
- **不构成竞争**：需要 GUI 组件，更接近 VSCode 而非终端编辑器
- 还在 pre-1.0，不够成熟

#### Micro（28K⭐）— 我们的宿主
- 优势：零依赖、GUI 风格快捷键、Lua 插件、鼠标支持
- 劣势：**开发停滞**，858 个 issue 堆积，1-2 commits/月
- 这恰恰是我们的机会——原版停滞，fork 激活

#### Leaf（1.4K⭐）— 最直接竞品
- 优势：Markdown 渲染漂亮，表格、LaTeX、Mermaid 都支持
- 劣势：**只能看不能编辑**
- 1人开发，53天从零到1.4K star，每天1个版本，增长很快
- 我们做的是 Micro + Leaf 的合体

#### Glow — Markdown 查看器
- charmbracelet 团队出品，Go 语言，约 3,500 行
- 交互体验不如 Leaf（没有 watch、搜索等）
- 最后更新 2026-04-22

## 四、Leaf 的成功经验

Leaf 值得学习的地方：

### 产品层面
- 一句话定位清晰："Terminal Markdown previewer — GUI-like experience"
- 53 天发了 54 个版本，平均每天 1 版
- 1.6 万行 Rust，一个人写的

### 宣传层面
- 精美的 Logo + SVG wordmark
- 11 张高质量截图，每个功能都有独立 demo
- 335 行的专业 README，像产品说明书
- 一行安装：curl、npm、AUR 全覆盖
- 蹭 AI 热点，专门写了 "AI Workflow" 章节
- 高频发版 = 持续出现在 GitHub Trending

### 技术层面
- 作者先造了依赖轮子（mmdflux、syntect），再搭 Leaf
- Rust 编译慢（1.6万行编译要30秒-1分钟），但终端渲染性能好

## 六、开源发布策略

### 许可证

Micro 使用 **MIT 许可证**，允许：
- ✅ Fork 代码
- ✅ 修改功能
- ✅ 改名发布
- ✅ 公开发布给大家用

**唯一要求**：保留原始 LICENSE 文件和版权声明。

### 发布路径

1. **阶段一：Fork + 自己用**
   - 在 GitHub 上 fork Micro
   - 加表格渲染功能
   - 自己编译自己用，验证好不好用

2. **阶段二：打磨 + 公开发布**
   - 代码写干净，加注释
   - 起个好名字（如 microneo）
   - 写 README，放 before/after 对比截图
   - 公开发布

3. **阶段三：回馈上游或独立发展**
   - A) 提 PR 给 Micro 原版
   - B) 继续独立发展，积累自己的用户和 star

### 宣传计划（学 Leaf）

- before/after 对比截图（表格显示效果）
- 一行安装命令
- 在 Micro 的 issue 里提一嘴（28K star 的用户基础）
- 蹭 AI + Markdown 的热点
