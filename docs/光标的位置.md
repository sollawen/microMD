## 原生 scrollUp 怎么算新 StartLine

`scrollUp(s, n)`（`softwrap.go:232`）= 从起点 `s` **向上**数 `n` 个**视觉行**，用 `SLoc{Line, Row}` 二维定位：

```go
for n > 0 {
    if n <= s.Row {
        s.Row -= n          // 目标还在当前行内 → 只改 Row
        n = 0
    } else if s.Line > 0 {
        s.Line--            // 跨到上一行
        n -= s.Row + 1      // 消耗掉当前行剩余的 (s.Row+1) 个视觉行
        s.Row = w.getRowCount(s.Line) - 1   // 跳到上一行的最后一个 row
    } else {
        s.Row = 0; break    // 到顶
    }
}
```

核心：**逐视觉行**走，跨行时用 `getRowCount(line)` 问"这一行占几个视觉行"。

## 新 StartLine 是 softwrap 长行怎么办 —— 正好是 Row 字段的用武之地

走个具体例子。光标 `c = {line:10, row:0}`，`botMarginRow=3`，要 `scrollUp(c, 3)`：

```
迭代1: s={10,0}, n=3。n(3) > s.Row(0) → 跨行。s={9,0}, n=2  (line9 普通行，1 row)
迭代2: s={9,0},  n=2。n(2) > s.Row(0) → 跨行。s={8,4}, n=1  (line8 长行，5 rows)
迭代3: s={8,4},  n=1。n(1) ≤ s.Row(4) → s.Row=3。完成。
→ StartLine = {line:8, row:3}
```

**结果 StartLine 落在 line 8 的第 3 个 softwrap 段**。不是行首，而是中间。

渲染时（`bufwindow.go:480`）：
```go
vloc.Y = -w.StartLine.Row    // = -3，把 line8 的前 3 个 row 滚出屏幕
```

line 8 的 row 0/1/2 滚到视口上方（不可见），从 row 3 开始画。**完美对齐，没有内容偏移**。

这就是 `SLoc.Row` 存在的根本理由：**让"视口顶部落在某长行的第 k 行"这种状态能被精确表达**。
